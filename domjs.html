Callbacks & Async Basics

Understanding asynchronous programming and the callback pattern.

What is Asynchronous JavaScript?

  • JavaScript is single-threaded but can handle async operations
  • Non-blocking operations: timers, network requests, file I/O
  • Callbacks were the original way to handle async code
  • Modern approaches: Promises and async/await

Callback Functions

// Simple callback example function fetchData(callback) { setTimeout(() => { const data = { name: "John", age: 30 }; callback(null, data); // Node.js convention: error first }, 1000); } // Using the callback fetchData((error, data) => { if (error) { console.error("Error:", error); } else { console.log("Data received:", data); } }); // Real-world example: Reading a file fs.readFile('file.txt', 'utf8', (err, content) => { if (err) { console.error("Error reading file:", err); return; } console.log("File content:", content); });

Callback Hell (Pyramid of Doom)

// Nested callbacks become hard to read getUser(userId, (err, user) => { if (err) { console.error("Error getting user:", err); return; } getPosts(user.id, (err, posts) => { if (err) { console.error("Error getting posts:", err); return; } getComments(posts[0].id, (err, comments) => { if (err) { console.error("Error getting comments:", err); return; } console.log("User:", user.name); console.log("First post:", posts[0].title); console.log("Comments:", comments.length); }); }); });
Callback Hell: Deeply nested callbacks are hard to read and maintain. This led to the creation of Promises.

JavaScript Promises

Promises provide a cleaner way to handle asynchronous operations.

What is a Promise?

  • An object representing the eventual completion of an async operation
  • Three states: pending, fulfilled, rejected
  • Chainable with .then() and .catch()
  • Better error handling than callbacks

Creating and Using Promises

// Creating a promise const myPromise = new Promise((resolve, reject) => { // Async operation here setTimeout(() => { const success = Math.random() > 0.5; if (success) { resolve("Operation completed successfully!"); } else { reject("Operation failed!"); } }, 1000); }); // Using the promise myPromise .then(result => { console.log("Success:", result); }) .catch(error => { console.error("Error:", error); }) .finally(() => { console.log("Operation completed (success or failure)"); }); // Promise that always resolves const immediatePromise = Promise.resolve("Immediate value"); // Promise that always rejects const failedPromise = Promise.reject("Immediate failure");

Promise Chaining

// Chaining multiple async operations fetchUser(userId) .then(user => { console.log("User:", user); return fetchPosts(user.id); // Return a new promise }) .then(posts => { console.log("Posts:", posts); return fetchComments(posts[0].id); }) .then(comments => { console.log("Comments:", comments); }) .catch(error => { console.error("Error in chain:", error); }); // Multiple parallel operations Promise.all([ fetchUser(userId), fetchPosts(userId), fetchSettings(userId) ]) .then(([user, posts, settings]) => { console.log("All data loaded:", { user, posts, settings }); }) .catch(error => { console.error("One of the promises failed:", error); });

Promise Static Methods

// Promise.all - wait for all promises to resolve Promise.all([promise1, promise2, promise3]) .then(values => { console.log("All resolved:", values); }) .catch(error => { console.error("One rejected:", error); }); // Promise.race - first settled promise wins Promise.race([fastPromise, slowPromise]) .then(value => { console.log("First to finish:", value); }); // Promise.allSettled - wait for all to settle Promise.allSettled([promise1, promise2]) .then(results => { results.forEach(result => { if (result.status === 'fulfilled') { console.log("Success:", result.value); } else { console.log("Failed:", result.reason); } }); }); // Promise.any - first fulfilled promise Promise.any([promise1, promise2]) .then(value => { console.log("First success:", value); });

Async/Await

Syntactic sugar for working with promises - makes async code look synchronous.

Async Functions

// Async function always returns a promise async function fetchUserData(userId) { // Even if you return a non-promise value, // it will be wrapped in a promise return { id: userId, name: "John" }; } // Equivalent to: function fetchUserData(userId) { return Promise.resolve({ id: userId, name: "John" }); } // Using async function fetchUserData(123) .then(user => console.log(user)) .catch(error => console.error(error));

Await Keyword

// await can only be used inside async functions async function getUserDetails(userId) { try { // Wait for the promise to resolve const user = await fetchUser(userId); console.log("User:", user); // Wait for another promise const posts = await fetchPosts(user.id); console.log("Posts:", posts); // Wait for multiple promises in parallel const [settings, preferences] = await Promise.all([ fetchSettings(user.id), fetchPreferences(user.id) ]); return { user, posts, settings, preferences }; } catch (error) { console.error("Error fetching user details:", error); throw error; // Re-throw the error } } // Using the async function getUserDetails(123) .then(details => console.log("All details:", details)) .catch(error => console.error("Failed:", error));

Error Handling with Async/Await

// Method 1: try/catch async function fetchData() { try { const response = await fetch('/api/data'); const data = await response.json(); return data; } catch (error) { console.error('Fetch failed:', error); // Return default value or re-throw return { default: true }; } } // Method 2: .catch() on the returned promise async function fetchData() { const response = await fetch('/api/data'); return await response.json(); } fetchData().catch(error => { console.error('Failed:', error); }); // Method 3: Handle errors individually async function robustFetch() { const response = await fetch('/api/data').catch(() => ({ ok: false, json: () => Promise.resolve({ error: 'Fetch failed' }) })); if (!response.ok) { return { error: 'Request failed' }; } return await response.json(); }

APIs & Fetch

Making HTTP requests and working with external APIs.

Fetch API Basics

// Basic GET request fetch('https://api.example.com/users') .then(response => { if (!response.ok) { throw new Error('Network response was not ok'); } return response.json(); // Parse JSON }) .then(data => { console.log('Users:', data); }) .catch(error => { console.error('Fetch error:', error); }); // With async/await async function getUsers() { try { const response = await fetch('https://api.example.com/users'); if (!response.ok) { throw new Error(`HTTP error! status: ${response.status}`); } const users = await response.json(); return users; } catch (error) { console.error('Failed to fetch users:', error); return []; } }

Different HTTP Methods

// POST request async function createUser(userData) { const response = await fetch('https://api.example.com/users', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer your-token-here' }, body: JSON.stringify(userData) }); if (!response.ok) { throw new Error('Failed to create user'); } return await response.json(); } // PUT request (update) async function updateUser(userId, userData) { const response = await fetch(`https://api.example.com/users/${userId}`, { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(userData) }); return await response.json(); } // DELETE request async function deleteUser(userId) { const response = await fetch(`https://api.example.com/users/${userId}`, { method: 'DELETE' }); if (!response.ok) { throw new Error('Failed to delete user'); } return true; // Success }

Error Handling and Best Practices

// Robust fetch wrapper async function robustFetch(url, options = {}) { const controller = new AbortController(); const timeoutId = setTimeout(() => controller.abort(), 5000); try { const response = await fetch(url, { ...options, signal: controller.signal }); clearTimeout(timeoutId); if (!response.ok) { throw new Error(`HTTP ${response.status}: ${response.statusText}`); } return await response.json(); } catch (error) { clearTimeout(timeoutId); if (error.name === 'AbortError') { throw new Error('Request timeout'); } throw error; } } // Usage with retry logic async function fetchWithRetry(url, retries = 3) { for (let i = 0; i < retries; i++) { try { return await robustFetch(url); } catch (error) { if (i === retries - 1) throw error; console.log(`Attempt ${i + 1} failed, retrying...`); await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1))); } } }
API Best Practices: Always handle errors, use timeouts, consider retry logic for unreliable networks, and be mindful of rate limits.
